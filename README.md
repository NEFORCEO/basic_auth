# Пользовательская система

Веб-приложение для управления пользователями с системой аутентификации.

## Описание

Это приложение предоставляет интерфейс для работы с пользователями, включая их регистрацию, аутентификацию и просмотр списка всех зарегистрированных пользователей.

## Основные возможности

- **Регистрация пользователей**: Добавление новых пользователей в систему
- **Аутентификация**: Проверка учетных данных пользователей
- **Просмотр пользователей**: Получение списка всех зарегистрированных пользователей
- **Система логирования**: Автоматическое логирование всех запросов к системе

## API Эндпоинты

### Регистрация/Аутентификация пользователя
```
GET /me/basic-auth
```
Регистрирует нового пользователя или проверяет существующего через базовую аутентификацию.

**Параметры:**
- Аутентификация через HTTP Basic Auth (username и password)

**Ответ:**
```json
{
  "status": 200,
  "username": "имя_пользователя",
  "password": "пароль"
}
```

### Получение списка пользователей
```
GET /me/basic_all
```
Возвращает список всех зарегистрированных пользователей в системе.

**Ответ:**
Список пользователей с их идентификаторами, именами и паролями.

## Как использовать

1. **Регистрация пользователя**: Отправьте GET запрос на `/me/basic-auth` с вашими учетными данными в заголовке аутентификации
2. **Просмотр пользователей**: Отправьте GET запрос на `/me/basic_all` для получения списка всех пользователей

## Безопасность

- Используется базовая аутентификация для защищенных операций
- Все запросы логируются для отслеживания активности
- Данные пользователей хранятся в защищенном формате

## Структура проекта

Проект организован по модульному принципу с разделением на следующие компоненты:

- **Маршрутизация** (`router/`): Определение API эндпоинтов
- **Модели данных** (`database/models/`): Структура пользователей
- **Схемы** (`schemas/`): Форматы запросов и ответов
- **База данных** (`database/database/`): Управление подключением и сессиями
- **Клиент** (`client/`): Конфигурация и запуск приложения
- **Логирование** (`loger/`): Система ведения журналов

## Примечания

- Приложение автоматически создает необходимые таблицы при запуске
- Все операции с базой данных выполняются асинхронно для повышения производительности
- Система логирования фиксирует все входящие запросы и ответы с временными метками
